## Affected Routes
- `GET /api/articles` — article list with filters (date_from, date_to, source, is_shown)
- `POST /api/articles` — manual article creation
- `PATCH /api/articles/{id}` — edit article (is_shown toggle, reason, headline)
- `DELETE /api/articles/{id}` — delete article
- `POST /api/crawl/run` — manual crawl trigger (202 async)
- `GET /health` — liveness check

## Key Interactions to Verify
- Article CRUD via frontend form modal (create, edit, delete)
- `is_shown` toggle in edit form updates DB and removes/adds card
- Crawl button click → 202 immediately → articles appear after ~5s
- Manual article always `is_shown=True`, `is_manual=True`
- Score=1 auto-article has Korean reason, appears in board
- Score=0 / score=-1 articles stored in DB but NOT visible in frontend

## Edge Cases
- Duplicate URL submission → 409 Conflict or graceful skip
- LLM API timeout → retry 3x with exponential backoff, skip article, log warning
- GDELT returns empty results → no crash, no articles added
- `POST /api/crawl/run` when FastAPI unreachable → Mock 202 response
- `reason` field is null for score≠1 articles → frontend handles null gracefully

## Critical Paths
- Full crawl cycle: GDELT fetch → dedup → LLM score → DB write → frontend display
- Manual add → immediate display on board (is_manual=True bypass scoring)
- `is_shown` toggle: auto-article scored 1 but user hides it → disappears from board

Run pytest with actual Claude API call, assert expected score matches.

## Test Framework
- Backend: pytest + httpx AsyncClient (FastAPI test client)
- DB: separate test PostgreSQL or SQLite in-memory for unit tests
- LLM scorer eval: real Claude API call with fixture articles (mark as `@pytest.mark.eval`)
